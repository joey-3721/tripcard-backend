# 地理编码服务成本分析

## 推荐方案：高德 + Nominatim 混合

### 为什么不用 Google？
1. **太贵**：超出免费额度后 $17/千次
2. **中国无法访问**：需要代理，增加复杂度
3. **个人开发负担重**：一旦用户增长，成本爆炸

### 推荐架构

```
中国地区 → 高德地图 API（免费 30万次/天）
其他国家 → Nominatim（免费但数据质量一般）
         → 可选：Mapbox（免费 10万次/月，质量比 Nominatim 好）
```

### 成本对比

| 方案 | 月成本 | 优点 | 缺点 |
|------|--------|------|------|
| 高德+Google | $1000+ | 全球最准 | 太贵，Google 被墙 |
| 高德+Nominatim | ¥0 | 完全免费 | 国外数据一般 |
| 高德+Mapbox | ¥0 | 国外数据较好 | Mapbox 超额后 $5/千次 |
| 高德+Photon(现状) | ¥0 | 完全免费 | 国外数据差 |

### 最佳实践

**阶段 1：个人开发/小规模**
- 中国：高德 API（免费）
- 国外：Nominatim（免费）
- 成本：¥0/月

**阶段 2：用户增长（1000+ DAU）**
- 中国：高德 API（免费）
- 国外：Mapbox（免费 10万次/月）
- 成本：¥0-500/月

**阶段 3：商业化（10000+ DAU）**
- 考虑自建 OSM 数据库
- 或者付费使用 Google Places
- 成本：¥5000+/月
